# Restaurant Data Analysis using SQL

## Project Overview
This project focuses on analyzing restaurant data using SQL to extract meaningful insights. The dataset contains information about restaurants such as city, cuisine, cost, rating, and rating count. 

Using SQL queries, the project explores various business questions like identifying top restaurants, revenue generation, popular cuisines, and city-wise restaurant distribution.

## Tools & Technologies
- SQL
- MySQL
- Data Analysis Techniques
- Window Functions
- Aggregate Functions
- Subqueries

## Dataset Features
The dataset includes the following attributes:

- Restaurant Name
- City
- Cuisine Type
- Cost
- Rating
- Rating Count

## Key Analysis Performed

### Basic SQL Queries
- Retrieve restaurant information
- Filter restaurants by city
- Find restaurants with high ratings
- Identify affordable restaurants
- List distinct cuisines

### Aggregation Analysis
- Average restaurant rating by city
- Average cost by cuisine
- Count of restaurants per city
- Maximum and minimum cost by cuisine

### Revenue Analysis
Revenue is calculated using:

Revenue = Cost × Rating Count

This helps determine which restaurants and cities generate the highest revenue.

### Business Insights Queries
Examples of insights extracted:

- Restaurant with maximum revenue in India
- Most visited restaurant in a city
- City with highest number of restaurants
- Most popular cuisine in each city
- Least expensive and most expensive cuisines

### Advanced SQL (Window Functions)

The project also uses advanced SQL techniques such as:

- `ROW_NUMBER()`
- `RANK()`
- `DENSE_RANK()`

These are used to:

- Rank restaurants based on cost
- Rank restaurants by visits
- Find top restaurants per city
- Find top restaurants per cuisine

Example analysis:

- Top 5 restaurants in each city based on revenue
- Ranking restaurants by price within each city
- Ranking restaurants by cuisine price
